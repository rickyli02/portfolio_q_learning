WHAT IS MISSING — BY TIER

Tier 1: Core algorithm correctness gaps (HJZ 2025 theorem requirements)
These are gaps between the current implementation and what the convergence proof requires.

1. Parameter projection onto expanding sets K_{θ,n}, K_{1,n}, K_{2,n}
   - w-projection exists (w_min/w_max bounds in w_update.py)
   - θ/φ parameter projection is ABSENT
   - The convergence theorem (HJZ 2025 §3-4) requires all three projection sets
   - Without it, the repo is a practical approximation, not the theorem-compliant algorithm

2. φ₂^{-1} (precision) update
   - The paper specifies updating φ₂^{-1} not φ₂ (numerical stability + convergence)
   - Actor currently stores log_phi2_inv and optimizes via autograd through it

   - Components exist: actor.sample() for behavior, actor.mean_action() for execution
   - The training loop (ctrl_trainer.py, ctrl_outer_iter.py) does not explicitly
     enforce or document which policy is used where
   - The paper requires stochastic behavior policy for data collection and
     deterministic execution policy for reported performance

Tier 2: Practical online variant (HJZ 2022 + E-companion)
These are the production-grade improvements the companion paper adds.

1. TD(λ) eligibility traces (I_k, H_k with exponential decay λ^{t-s})
   - Not implemented; currently uses full batch accumulation per outer iteration

2. One-step incremental gradient updates G_{θ,k}, G_{φ₁,k}, G_{φ₂^{-1},k}
   - Not implemented; currently runs n_updates full gradient steps per outer iter

3. Historical mini-batch weighting: θ_{k+1} ← Proj(θ_k + a_n(w_prev·G_{k-1} + w_curr·G_k))
   - Not implemented; plain single-step Adam/SGD updates

4. Separate rebalance cadence R from parameter update cadence U
   - Not implemented; currently each env step maps 1:1 to a gradient step

5. Leverage constraint layer
   - Risky-only projection exists: src/envs/gbm_env.py (apply_risky_only_projection)
   - Leverage limit (rescale if ∑|u| > x·ℓ) is ABSENT

6. Off-policy learning seam (explicit separation of data-generating vs eval policy)
   - Not implemented as a named seam
